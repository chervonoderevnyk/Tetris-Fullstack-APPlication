# Політика Безпеки Паролів - Tetris Game

## Огляд

Проект Tetris Game тепер має посилені вимоги до безпеки паролів для захисту облікових записів користувачів та забезпечення безпеки системи.

## Вимоги до Пароля

### Обов'язкові Критерії

1. **Мінімальна довжина**: 8 символів (раніше було 6)
2. **Максимальна довжина**: 100 символів
3. **Велика літера**: Принаймні одна (A-Z)
4. **Мала літера**: Принаймні одна (a-z)
5. **Цифра**: Принаймні одна (0-9)
6. **Спеціальний символ**: Принаймні один (!@#$%^&*()_+-=[]{};':"\\|,.<>/?~`)

### Заборони

1. **Поширені паролі**: Система блокує використання популярних слабких паролів:
   - password, password123, 123456, qwerty, abc123 тощо

2. **Схожість з іменем користувача**: 
   - Пароль не може містити ім'я користувача
   - Ім'я користувача не може бути частиною пароля

3. **Повторювані символи**: 
   - Заборонено 3+ однакових символи підряд (aaa, 111)

4. **Послідовні символи**: 
   - Заборонено послідовності типу 123, abc, def

## Рівні Міцності

Система оцінює пароль за шкалою 0-100 балів:

- **Слабкий (0-39)**: Червоний індикатор ❌
- **Середній (40-69)**: Помаранчевий індикатор ⚠️  
- **Сильний (70-100)**: Зелений індикатор ✅

### Фактори Оцінки

**Позитивні:**
- Базові вимоги (+15 балів кожна)
- Довжина 8+ символів (+20 балів)
- Довжина 12+ символів (+10 бонус)
- Висока різноманітність символів (+10 бонус)

**Негативні:**
- Поширений пароль (максимум 30 балів)
- Повторювані шаблони (-10 балів)
- Послідовні символи (-10 балів)

## API Endpoints

### Перевірка Міцності Пароля
```http
POST /auth/check-password-strength
Content-Type: application/json

{
  "password": "MyStr0ng!Password",
  "username": "testuser" // опціонально
}
```

**Відповідь:**
```json
{
  "isValid": true,
  "errors": [],
  "strength": "strong",
  "score": 85,
  "tips": [
    "Використовуйте принаймні 8 символів",
    "Комбінуйте великі та малі літери",

  ]
}
```

## Frontend Компоненти

### PasswordStrengthComponent

Компонент відображає:
- Індикатор міцності з кольоровою шкалою
- Список вимог з позначками ✓/✗
- Помилки валідації в реальному часі
- Поради для покращення
- Генератор безпечних паролів

**Використання:**
```html
<app-password-strength 
  [password]="password"
  [username]="username"
  [showTips]="true"
  [showGenerator]="true"
  (passwordGenerated)="onPasswordGenerated($event)">
</app-password-strength>
```

## Службі

### PasswordValidationService

**Методи:**
- `checkPasswordStrength()` - Перевіряє міцність через API
- `validatePasswordRequirementsLocal()` - Локальна швидка перевірка
- `generateSecurePassword()` - Генерує безпечний пароль
- `getStrengthColor()` - Повертає колір індикатора
- `getStrengthText()` - Повертає текст міцності

### AuthService

Оновлений для валідації паролів при реєстрації:
- Автоматична перевірка міцності
- Перевірка схожості з іменем користувача
- Відображення детальних помилок

## Приклади Паролів

### ✅ Сильні Паролі
- `MyStr0ng!Password`
- `C0mpl3x#Pass2024`
- `Secure&P@ssw0rd`
- `Ungu3ss@bl3!Pwd`

### ❌ Слабкі Паролі
- `password` (поширений)
- `Password123` (без спеціальних символів)
- `testuser123` (містить ім'я користувача)
- `Passsss1!` (повторювані символи)
- `Pass123!` (послідовні символи)

## Безпека Реалізації

1. **Хешування**: Усі паролі хешуються з bcrypt
2. **Валідація на сервері**: Подвійна перевірка backend + frontend
3. **Захист від атак**: Rate limiting для API endpoints
4. **Логування**: Відстежування спроб входу та реєстрації
5. **HTTPS**: Обов'язково для продакшн середовища

## Налаштування

Конфігурація в `.env`:
```env
BCRYPT_ROUNDS=10
```

Мінімальна довжина налаштовується в `password-validator.ts`:
```typescript
private static readonly MIN_LENGTH = 8;
```

## Міграція Існуючих Паролів

Користувачі з паролями, що не відповідають новим вимогам:
1. Можуть продовжувати використовувати існуючий пароль
2. При зміні пароля повинні дотримуватися нових правил
3. Система показує рекомендації щодо оновлення пароля
